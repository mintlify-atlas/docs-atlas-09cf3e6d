BestClient is a custom DDNet client forked from TClient (Tater), built around native BestClient systems rather than a thin reskin of the vanilla experience. Where standard DDNet clients expose a handful of cosmetic toggles, BestClient ships its own rendering stack, input tooling, social systems, full-screen editors, and an in-client content flow — all while remaining a first-class citizen of the DDNet ecosystem. Version **1.9.1 beta** (build `101`, dated 28.05) is the current release, carrying 280 `bc_*` config entries, 52 component toggles, a dedicated ReShade tab, integrated voice and media systems, live HUD tooling, an in-client asset shop, and a curated Fun tab.

<Note>
  BestClient connects to standard DDNet servers and is fully compatible with all DDNet maps, game modes, and server-side features. No special server software is required.
</Note>

## What Makes BestClient Different

Vanilla DDNet is an excellent base client, but it was designed for broad compatibility rather than competitive comfort or deep customization. BestClient addresses this by replacing or extending four core areas:

- **Native visual stack** — effects like Camera Drift, Dynamic FOV, Motion Blur, Afterimage, and particle systems are implemented as first-party engine features, not post-process hacks.
- **Input system** — Fast Input, Delta Input, Best Input, and Saiko+ give players granular control over how mouse and keyboard events are processed, reducing perceived latency and improving consistency.
- **Social systems** — integrated Voice Chat with VAD, Chat Media previews, a Client Indicator, and streamer helpers make the in-game communication layer richer without external tools.
- **Editors and tooling** — the Assets Editor, Components Editor, and HUD Editor provide fullscreen native UIs for managing client modules, rather than burying toggles in a scrolling settings list.
